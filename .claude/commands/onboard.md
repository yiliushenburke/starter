Summarize architecture, standards, and state to onboard a new developer.

**Step 1: Deep Analysis**
Explore the following:
1. **Project Structure:** Main directories (src, tests, docs) and their purposes.
2. **Core Architecture:** Primary technologies, frameworks, and component interactions.
3. **Coding Standards:** Read `CLAUDE.md`, `GEMINI.md`, `README.md`, `DESIGN.md`, `makefile`, and `pyproject.toml` for workflow standards (TDD, linting, dependency management).
4. **Current State:** Read `CHANGELOG.md` and latest `journal/` entries for recent changes.
5. **Test Suite:** How tests are structured and how to run them.

**Step 2: Generate the Onboarding Guide**
Produce a well-structured report with:
- **Project Overview:** High-level summary of what the project does.
- **Architecture & Core Components:** The "brain" of the project and data flow.
- **Getting Started:** Essential commands to set up, run tests, and start the application.
- **Coding Standards:** Key rules for documentation, testing, and style.
- **Current Focus:** What the team has been working on (from journal/changelog).
- **Entry Points:** The most important files to read first.

Use a welcoming and technically rigorous tone. Suggest follow-up questions for deeper exploration.
